Executive Summary
We propose a Digital Arrest Risk Scoring (DARS) Graph Neural Network (GNN) pipeline that integrates transaction-level anomaly detection with network-based fraud analysis. The pipeline ingests two input datasets: (1) UPI transaction data (CSV) and (2) bank/account transaction statements (XLSX), and performs end-to-end processing, modeling, and visualization. Major steps include:
Data Loading & Cleaning: Read UPI and bank data, handle missing values, parse dates, and normalize formats. For example, an Isolation Forest requires no missing values, so we impute or drop as neededπÇÉ5ΓÇáL303-L311πÇæ.
Feature Engineering: Compute behavioral features (e.g., a userΓÇÖs typical transaction amount and timing), geographic risk (based on sender location), and vulnerability factors (e.g., age group). Adding features like ΓÇ£hour of dayΓÇ¥ or ΓÇ£transaction typeΓÇ¥ can greatly enhance anomaly detectionπÇÉ5ΓÇáL309-L312πÇæ. We also address class imbalance: since fraud is rare, we use SMOTE oversampling to ensure sufficient fraud samplesπÇÉ28ΓÇáL85-L86πÇæ.
Isolation Forest: We train a sklearn IsolationForest to assign a Behavioral Risk Score (BRS) to each transaction. Isolation Forest is known to isolate anomalies via random splits, making it effective for fraud outliersπÇÉ5ΓÇáL233-L241πÇæ.
Graph Construction & GAT: We construct a directed transaction graph from the bank data (nodes = accounts, edges = fund transfers). Node features include total deposits, withdrawals, transaction count, etc. We then train a Graph Attention Network (GAT) using PyTorch Geometric to learn node embeddings. GAT assigns learned attention weights to neighbors, so each node ΓÇ£listensΓÇ¥ more to suspicious connectionsπÇÉ18ΓÇáL11-L16πÇæπÇÉ19ΓÇáL1-L4πÇæ. The output node embeddings are used to compute a Scam Network Score (SNS) (e.g., via embedding norms or a trained detector).
Shell Account Classifier (XGBoost): Using node embeddings and engineered features, we train an XGBoost classifier to identify likely shell/mule accounts (Shell Account Score, SAS). XGBoostΓÇÖs scikit-learn API provides a convenient interface for binary classificationπÇÉ26ΓÇáL78-L86πÇæ. We calibrate for imbalance by setting scale_pos_weight or using SMOTE.

Evaluation & Visualization: The pipeline computes evaluation metrics (confusion matrix, precision/recall/F1, ROC AUC) for fraud detection. We generate publication-quality figures: risk score histograms, feature-importance plots (using SHAP for XGBoostπÇÉ13ΓÇáL325-L333πÇæ), fraud vs. normal transaction comparisons, and network diagrams of suspicious account clusters. Feature Engineering
We derive the following features and transformations:
Behavioral Features: For each sender in the UPI data, we compute historical statistics (mean/median amount, transaction count, frequency per hour, favorite merchant categories, etc.) to detect anomalies. For example, adding features like time of day and transaction type helps identify off-hours anomaliesπÇÉ5ΓÇáL309-L312πÇæ.
Geographic Risk (GRS): We compute geographic deviation: e.g. if a sender normally transacts within one state but this txn is out-of-state, we set a high Geographic Risk. (If latitude/longitude were available, one could compute physical distance; with only state-level data, we use a simple penalty for state mismatch.)
Vulnerability (VA): We assign higher risk if the sender is in a known target demographic: for instance, age ΓÇ£56+ΓÇ¥ or inexperienced device users. This is a heuristic user profile score.
One-Hot Encoding: Categorical fields (transaction type, merchant category, banks, network) are one-hot or label-encoded.
Temporal Features: Already provided hour_of_day, day_of_week, is_weekend allow capture of unusual timings (e.g. midnight transfers).
Imbalance Handling: We use SMOTE to oversample the minority class (fraud) before training classifiersπÇÉ28ΓÇáL85-L86πÇæ. This ensures the model has enough fraud examples to learn meaningful patterns.
Graph Features (Node Attributes): For each account node (from bank data), we compute features like total deposits, total withdrawals, transaction count, and net flow ratio. We also include static features (e.g. account tenure = days active) where possible.
# features.pyimport pandas as pdimport numpy as npfrom sklearn.preprocessing import OneHotEncoder, LabelEncoderfrom imblearn.over_sampling import SMOTEdef compute_behavioral_features(df):    """    Compute user-behavior features: historical mean and std of amount,     transactions per hour, etc., grouped by sender.    """    # Example: aggregate statistics by sender bank & age group (proxy for user)    agg = df.groupby(['sender_bank','sender_age_group']).agg(        tx_count=('transaction id','count'),        avg_amount=('amount (INR)', 'mean'),        std_amount=('amount (INR)', 'std'),        unique_receivers=('receiver_bank', pd.Series.nunique)    ).reset_index()    agg.columns = ['sender_bank','sender_age_group','tx_count','avg_amt','std_amt','uniq_rec']    df = df.merge(agg, on=['sender_bank','sender_age_group'], how='left')    # Feature: is current amount > 3 * historical mean?    df['high_amount_flag'] = (df['amount (INR)'] > 3 * df['avg_amt']).astype(int)    return dfdef compute_geographic_risk(df):    """    Simple geographic risk: if current state not in sender's historical states, flag high risk.    """    # historical sender states    hist_states = df.groupby(['sender_bank','sender_age_group'])['sender_state'].agg(set).reset_index()    hist_states.columns = ['sender_bank','sender_age_group','state_set']    df = df.merge(hist_states, on=['sender_bank','sender_age_group'], how='left')    df['GRS'] = np.where(df.apply(lambda row: row['sender_state'] not in row['state_set'], axis=1), 100, 0)    return dfdef compute_vulnerability(df):    """    Compute vulnerability score based on age group.    """    # Example: higher weight for older age groups    age_map = {'18-25': 0.2, '26-35': 0.1, '36-45': 0.1, '46-55': 0.2, '56+': 0.4}    df['VA'] = df['sender_age_group'].map(age_map).fillna(0.1)    return dfdef encode_categoricals(df, columns):    """    One-hot encode specified categorical columns.    """    encoders = {}    for col in columns:        le = LabelEncoder()        df[col+'_enc'] = le.fit_transform(df[col].fillna('Unknown'))        encoders[col] = le    return df, encodersdef resample_smote(X, y):    """    Apply SMOTE to balance the classes.    """    sm = SMOTE(random_state=42)    X_res, y_res = sm.fit_resample(X, y)    return X_res, y_res

Data Loading and Preprocessing
We implement a data_loader.py that reads the CSV/XLSX, normalizes formats, and outputs clean DataFrames. Key steps:
UPI CSV: Use pandas.read_csv(), parse dates, and drop or impute missing values. Ensure types (int, datetime) are correct.
Bank XLSX: The Excel is structured with repeated account blocks. We read it into a single DataFrame and then split by Account No. We parse the TRANSACTION DETAILS to create directed edges: e.g., if it contains "TRF FROM XXX", we create an edge (XXX -> this_account). If "TRF TO YYY", edge (this_account -> YYY). We also record deposit/withdrawal amounts.
Cleaning: Standardize date columns to datetime, strip stray characters (e.g. the trailing ' in account numbers), rename columns to code-friendly names, and remove any helper columns (like the unnamed trailing ΓÇ£.ΓÇ¥ column).
# data_loader.pyimport pandas as pdimport numpy as npdef load_upi_data(csv_path):    """    Load and preprocess the UPI transactions CSV.    """    df = pd.read_csv(csv_path, parse_dates=['timestamp'])    # Standardize column names    df.columns = df.columns.str.strip().str.replace(' ', '_').str.replace('(', '').str.replace(')', '')    # Drop irrelevant or empty columns if any    # Handle missing values: e.g. drop rows with missing essential data    df = df.dropna(subset=['transaction_id','timestamp','amount_INR'])    return dfdef load_bank_data(xlsx_path):    """    Load and preprocess the bank statement XLSX.    Returns a concatenated DataFrame of all transactions.    """    df = pd.read_excel(xlsx_path, dtype=str)    df.columns = df.columns.str.strip()    # Remove trailing columns if any (the ' . ' column)    if '.' in df.columns:        df = df.drop(columns='.')    # Remove rows that are all NaN    df = df.dropna(how='all')    # Fill blank Account No with forward fill (each block starts with account)    df['Account No'] = df['Account No'].ffill()    return dfdef construct_transaction_graph(bank_df):    """    Parse bank transactions to create edge list for graph.    Returns list of edges (src, dst) and node features map.    """    edges = []    node_features = {}    for _, row in bank_df.iterrows():        acc = row['Account No'].strip("'")  # remove stray quotes        details = str(row['TRANSACTION DETAILS']).upper()        if 'TRF FROM' in details:            parts = details.split()            # Account after "TRF FROM"            src = parts[-1]  # naive: last part is sender            dst = acc        elif 'TRF TO' in details:            parts = details.split()            src = acc            dst = parts[-1]   # receiver account        else:            # Skip or handle IMPS, NEFT differently            continue        edges.append((src, dst))        # Collect feature: treat deposit as inflow, withdrawal as outflow        amt = float(row.get('DEPOSIT AMT') or 0.0)        if amt>0:            node_features.setdefault(dst, {})['total_deposit'] = node_features.get(dst, {}).get('total_deposit',0) + amt        amt2 = float(row.get('WITHDRAWAL AMT') or 0.0)        if amt2>0:            node_features.setdefault(src, {})['total_withdrawal'] = node_features.get(src, {}).get('total_withdrawal',0) + amt2    return edges, node_features
References: Data preprocessing must ensure quality inputs for anomaly modelsπÇÉ5ΓÇáL303-L311πÇæ. Pandas is used for ETL, and transactions are split into source/destination accounts when parsing descriptions. In practice, error handling and logging should be added for robustness.
Model Components
Isolation Forest (Behavioral Risk)
We train an IsolationForest on transaction feature vectors to score anomalies. We use scikit-learnΓÇÖs implementation, which isolates anomalies via random partitionsπÇÉ5ΓÇáL233-L241πÇæ. Hyperparameters like n_estimators=100, max_samples='auto', and a tuned contamination are specified.
# models.py (excerpt)from sklearn.ensemble import IsolationForestimport numpy as npclass BehavioralAnomalyModel:    """    Wrapper for Isolation Forest to score transaction anomalies.    """    def __init__(self, n_estimators=100, contamination=0.002, random_state=42):        self.model = IsolationForest(n_estimators=n_estimators,                                     max_samples='auto',                                     contamination=contamination,                                     random_state=random_state)    def fit(self, X):        self.model.fit(X)    def score(self, X):        # IsolationForest: lower score = more anomalous        scores = -self.model.decision_function(X)        return scores  # Higher = more anomalous# Training (example):# X_brs = features after scaling# iso_model = BehavioralAnomalyModel()# iso_model.fit(X_brs)# brs_scores = iso_model.score(X_brs)
Citation: Isolation Forest is widely used in financial fraud detection due to its effectiveness on high-dimensional transactional dataπÇÉ5ΓÇáL233-L241πÇæ. It produces an anomaly score (here we invert it so that higher = more anomalous) that we normalize into [0,100] for BRS.
Graph Attention Network (GAT) for Scam Network Score
We build a graph where nodes = bank accounts and edges = fund transfers (from the bank statements). Each node is represented by its engineered features (deposits, withdrawals, etc). We use PyTorch GeometricΓÇÖs GATConv layer to learn node embeddings. GAT assigns attention weights to neighbors, capturing which connected accounts are more ΓÇ£suspiciousΓÇ¥πÇÉ18ΓÇáL11-L16πÇæπÇÉ19ΓÇáL1-L4πÇæ. We stack a couple of GAT layers for deep representation.
# models.py (continued)import torchfrom torch import nnimport torch.nn.functional as Ffrom torch_geometric.nn import GATConvclass GATNetwork(nn.Module):    """    A two-layer GAT for node classification/embedding.    """    def __init__(self, in_channels, hidden_channels=64, heads=4, dropout=0.1):        super().__init__()        self.conv1 = GATConv(in_channels, hidden_channels, heads=heads, dropout=dropout)        self.conv2 = GATConv(hidden_channels*heads, hidden_channels, heads=1, concat=False, dropout=dropout)        self.dropout = dropout    def forward(self, x, edge_index):        x = self.conv1(x, edge_index)        x = F.elu(x)        x = F.dropout(x, p=self.dropout, training=self.training)        x = self.conv2(x, edge_index)        return x  # node embeddings# Example usage (in train.py or similar):# data = torch_geometric.data.Data(x=node_features_tensor, edge_index=edge_index)# model = GATNetwork(in_channels=data.num_features).to(device)# optimizer = torch.optim.Adam(model.parameters(), lr=0.005)# for epoch in range(epochs): #     model.train()#     out = model(data.x, data.edge_index)#     # In absence of labels, we could use out as embeddings directly#     # (Alternatively, if we had labeled fraud accounts, we would compute a supervised loss here.)
References: We use PyTorch GeometricΓÇÖs official GATConv implementation, which realizes the attention mechanism described in the original Graph Attention Networks paperπÇÉ22ΓÇáL98-L105πÇæ. GATΓÇÖs learned attention allows the model to weigh edges differently: e.g., suspicious links get higher weightπÇÉ18ΓÇáL11-L16πÇæπÇÉ19ΓÇáL1-L4πÇæ. In fraud scenarios, this helps identify collusive networks. (Indeed, recent studies find that GNNs outperform RF/XGBoost when relational patterns matterπÇÉ15ΓÇáL43-L48πÇæ.) For large graphs, neighbor sampling is recommendedπÇÉ1ΓÇáL154-L163πÇæ.
After training (or inference), we derive each nodeΓÇÖs Scam Network Score (SNS). For simplicity, here we compute SNS as the L2-norm of the nodeΓÇÖs embedding (or any monotonic function of it) ΓÇô nodes with atypical embeddings (far from normal clusters) get higher SNS.
# post-GAT score computation (in evaluate.py)import torchdef compute_sns_from_embeddings(embeddings):    """    Example: use L2 norm of embeddings as anomaly score (higher=more suspicious).    """    sns = torch.norm(embeddings, p=2, dim=1)    # Scale to 0-100    sns = 100 * (sns - sns.min()) / (sns.max() - sns.min() + 1e-6)    return sns.cpu().numpy()
XGBoost Shell Account Classifier
We train an XGBoostClassifier to label nodes (accounts) as shell/mule or not. The features include the GAT embeddings plus any additional account-level features. Using XGBoostΓÇÖs sklearn API, we set parameters (max_depth, n_estimators, etc.) and optionally use early stoppingπÇÉ26ΓÇáL78-L86πÇæ. The training labels could come from known fraud cases or heuristics (e.g., near-equal deposit/withdrawal). Here we outline the code; in practice, labeled data is needed.
# models.py (continued)import xgboost as xgbclass ShellAccountModel:    """    XGBoost model to classify shell accounts (binary). Combining into DARS
The final Digital Arrest Risk Score (DARS) for each transaction is a weighted sum of sub-scores:

We normalize each component to [0,100] and choose weights (╬▒,╬▓,╬│,╬┤) that reflect importance. For instance, we might set ╬▒=0.40, ╬▓=0.15, ╬│=0.35, ╬┤=0.10 (summing to 1). Prior work often scales each model output to [0,1] and combines via a weighted sumπÇÉ8ΓÇáL25-L28πÇæ. In a real deployment, these weights can be learned: one could train a logistic regression on known fraud labels using BRS, GRS, SNS, VA as features to optimize the combination.
# train.py (score combination)def combine_scores(brs, grs, sns, va, weights=None):    """    Compute final DARS as weighted sum of components.    """    # Normalize components 0-1    brs = (brs - brs.min()) / (brs.max() - brs.min() + 1e-9)    grs = (grs - grs.min()) / (grs.max() - grs.min() + 1e-9)    sns = (sns - sns.min()) / (sns.max() - sns.min() + 1e-9)    va  = (va - va.min())   / (va.max()  - va.min()  + 1e-9)    if weights is None:        # default weights        weights = {'alpha':0.4, 'beta':0.15, 'gamma':0.35, 'delta':0.10}    dars = (weights['alpha']*brs + weights['beta']*grs +            weights['gamma']*sns + weights['delta']*va)    # Scale to 0-100 for interpretability    dars = 100 * dars    return dars
Illustration: In one hybrid fraud study, model outputs (from XGBoost, Autoencoder, Isolation Forest, etc.) were first normalized to [0,1], then combined via a weighted sumπÇÉ8ΓÇáL25-L28πÇæ. We follow a similar principle for DARS.
